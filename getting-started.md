# Copilot Workspace の始め方

Copilot Workspace の技術プレビューへようこそ！ 👋 始めるためのいくつかの方法を以下に示します：

1. __GitHub リポジトリで問題を開き、「Open in Workspace」ボタンをクリックします。__ これにより、新しい Copilot Workspace セッションが開始され、タスクとして問題が事前に設定され、仕様 / 計画 / 実装を繰り返し行うことができます。

   <img src="images/general/open-in-workspace.png" width=800 alt="Copilot Workspace で問題を開く">

1. __統合ターミナルを開きます。__ ヘッダーバーのターミナルアイコンをクリックするか、「実装」パネルで「Open terminal」ボタンをクリックして、変更をビルド / テスト / 実行します（タスクの実装を開始した後）。

1. __セッションスナップショットを誰かと共有します。__ ヘッダーバーの右上にある共有ボタンをクリックし、URL を送信します。リポジトリにアクセスできる人は誰でもスナップショットの読み取り専用バージョンを表示できます。ただし、プレビューに参加している他の人にリンクを送信すると、セッションをフォークしてさらに探索できます。

1. __GitHub モバイルアプリをインストールします。__ 携帯電話にインストールし、「...」メニューをクリックして「Open in Workspace」を選択して問題を開きます。モバイル用に最適化された Copilot Workspace のモバイルウェブクライアントが表示されます。セッション / 編集は自動的に保存されるため、携帯電話でタスクを開始し、デスクトップで再開できます。

1. [Copilot Workspace ダッシュボード](https://copilot-workspace.githubnext.com) にアクセスし、「New Session」ボタンをクリックして__新しいセッションを開始します。__ これにより、リポジトリを検索し、アドホックタスクを定義できます。事実上、ドラフトの問題のようなものです。また、[テンプレートリポジトリ](https://docs.github.com/en/repositories/creating-and-managing-repositories/creating-a-template-repository) を選択すると、新しいリポジトリの要件を定義できます。

   <img src="images/general/dashboard.png" width=800 alt="Copilot Workspace で問題を開く">

1. __ワークスペースセッションを Codespace で開きます。__ ヘッダーバーまたは「実装」パネルで「Open in Codespace」ボタンをクリックします。ワークスペースの編集内容は Codespace に同期され、Codespace で行った編集もワークスペースに同期されます。これにより、VS Code / Codespaces を使用して大規模な編集やデバッグを行うことができます。

1. [アドホックタスク](adhoc-tasks.md) について学びます。アドホックタスクは、問題やプルリクエストのコンテキストなしに自分で定義するタスクです。Copilot Workspace ダッシュボードまたは GitHub.com のリポジトリページからアドホックタスクを開始できます。
